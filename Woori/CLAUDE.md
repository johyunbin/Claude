# Woori — 설교 녹취 · 교정 · 목장 나눔 자료

우리들교회 주일 설교를 **녹취 → 교정 → 목장 나눔 자료(PDF)** 로 만드는 작업 공간.
매주 같은 포맷으로 반복한다. `~/Claude` 종합 repo 안의 한 작업 영역.

## 작업 흐름 — 설교 1편당 3단계

### 1단계 · 녹취 (라이브 → 오디오 → transcript)

- 유튜브 라이브를 `yt-dlp`로 캡처하고 ffmpeg로 오디오만 추출 → `.m4a`.
- 다시보기가 안 남는 라이브는 방송 진행 중에 캡처해야 한다.
- 녹화 스크립트 `record_live.sh` — 안의 `URL` 값을 해당 설교 라이브로 바꾼 뒤
  `caffeinate -i bash "Woori/record_live.sh"` 로 백그라운드 실행. 종료는 `pkill -INT -f <video-id>`.
- 포맷 확인은 `yt-dlp -F <URL>`. 오디오 전용 트랙이 없으면 가장 낮은 화질의
  AAC-LC 묶음 포맷(예: 93번 360p)을 받아 `ffmpeg -vn -c:a copy`로 오디오만 추출한다.
- 완성된 `.m4a`를 다글로(daglo.ai)에 올려 STT transcript를 받는다. 다글로 길이 제한은 4시간.

### 2단계 · 교정 (transcript 교정)

- 다글로 STT 결과에서 **설교 구간만** 추린다(앞뒤 찬양·기도·광고 제외 — 타임스탬프로 경계 확인).
- 교정 지침은 `~/Documents/Claude/Projects/THINK 양육/CLAUDE.md` 의 「2. 설교 녹취 수정」을 따른다.
- STT 빈출 오류: 인명·성경 용어 대량 오번역(예: 므낫세→"문화세", 히스기야→"희숙이", 열왕기하→"11기야", 회개→"회계"). 개역개정과 대조해 인명·지명·구절을 바로잡고 비문을 정리한다.
- 결과물: `transcript 원본.txt`(다글로 원본 보존) + `transcript 교정.md`(교정 완료본).

### 3단계 · 목장 나눔 자료 (.md → PDF)

교정본을 바탕으로, 목장에서 목원들과 **함께 보며 진행**할 자료를 만든다.

**작업 순서 — 반드시 지킨다:**
1. `목장 나눔.md` 작성 (아래 「자료 구성」 템플릿 따름)
2. **심층 리뷰** — 오탈자, 성경 사실·인용의 정확성, 내용 완결성, 논리 흐름을 여러 각도로 점검·수정
3. PDF 변환: `python3 Woori/md2pdf.py "<설교 폴더>/목장 나눔.md"`
4. **전체 페이지 점검** — 생성된 PDF의 모든 페이지를 이미지로 직접 확인한다. 빈 페이지, 잘못된 페이지 나눔, 상단 정렬 불일치, 내용이 어중간하게 잘리는 곳을 점검
5. 문제가 있으면 수정 후 3~4단계 반복
6. 완성본을 iCloud Drive 루트에 복사(`목장 나눔 - <제목>.pdf`)해 사용자에게 전달

## 목장 나눔 자료 — 구성 (PDF 6쪽 템플릿)

| 쪽 | 내용 |
|----|------|
| 1 | 제목 · 도입 · **본문**(개역개정 전문) |
| 2 | **본문 이해** — 시대 배경 / 두 왕 대조 **표** / 핵심 단어 / 구속사 |
| 3 | **설교 한눈에**(세 대지 요약 **표**) + **대지 1** |
| 4 | **대지 2** |
| 5 | **대지 3** |
| 6 | 마무리 · 이번 한 주 삶 나눔 · 함께 기도 |

- **각 대지는 적용 질문(`함께 나눌 질문`)까지 한 페이지**에 담는다.
- **대지 구성**: `## 대지 N` 제목 + `<div class="key">핵심 — …</div>` 박스 + 굵은 머리말 단락 5~6개 + `#### ▷ 함께 나눌 질문`(3문항).
- **시각 요소**: 두 왕 대조 표, 세 대지 요약 표, 대지별 `<div class="key">` 핵심 박스, 성경 본문 인용(`>` 블록).
- **페이지 나눔**: `<div class="page-break"></div>` 를 `본문 이해`·`설교 한눈에`·`대지 2`·`대지 3`·`마무리` 앞에 둔다. (대지 1은 설교 한눈에 뒤에 이어지므로 나눔 없음.)

## md2pdf.py — PDF 변환기

- Trading `md2pdf`의 Chrome CDP 기법을 가져옴: 마크다운 → HTML → Chrome 헤드리스(CDP) → PDF.
- CSS: A4, 여백 12mm(모바일 가독성 고려), 본문 11pt·줄간격 1.6, 네이비(`#1f3a5c`)·크림 톤. 표·`.key` 박스·`.page-break` 스타일 포함.
- 의존성: `markdown`, `websocket-client`(pip), Google Chrome.
- ⚠ 여백은 CSS `@page { margin }` 으로 잡는다 — CDP 인쇄여백 파라미터만으로는 `@page` 설정에 덮어써진다.
- 생성된 `.pdf` 는 git에 commit하지 않는다(생성물). 배포는 iCloud Drive.

## 공통 규칙

| 항목 | 기준 |
|------|------|
| 성경 번역 | 개역개정 |
| 인명·지명 | 성경 표기 우선 |
| 우선순위 | 내용(성경 해석·설교 설명)의 정확성이 최우선. 인명 등 부수 정보의 정밀도는 부차적 |
| 출력 | 내용은 `.md`, 배포본은 `md2pdf.py`로 PDF 변환 |
| git 제외 | 오디오(`*.m4a`/`*.aac`)·PDF(`*.pdf`) — 용량 큰 생성물, 배포는 iCloud |
