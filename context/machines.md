## 머신 (AX3000BCM LAN — 라우터에 고정 IP 예약)

| 머신   | 고정 IP        | Tailscale IP      | OS           | 비고 |
|--------|----------------|-------------------|--------------|------|
| 맥미니 | 192.168.0.75   | 100.85.223.63     | macOS 26.5   | 메인. en0 **유선**(상위 공유기)으로 인터넷 + en1 Wi-Fi로 AX3000BCM |
| 맥북   | 192.168.0.25   | 100.127.92.10     | macOS 26.5   | MacBook Pro 14 |
| 그램   | 192.168.0.50   | 100.118.239.128   | Windows 11   | LG Gram. Wi-Fi 연결. 사용자 프로필 `wh850` |

- 같은 LAN(192.168.0.x)이므로 머신 간 직접 통신 가능. Tailscale로도 접근 가능.
- DHCP 예약이라 IP 안정적. 그래도 머신 간 접속은 Tailscale 100.x 주소가 가장 안전.

## 네트워크 토폴로지 (더블 NAT)

```
LG U+ 모뎀 (100M 회선)
  └─ 상위 공유기 192.168.123.1
       ├─ 맥미니 en0 (192.168.123.105) ── 맥미니 인터넷 경로 ★
       └─ AX3000BCM WAN (192.168.123.106)
            └─ AX3000BCM LAN 192.168.0.1
                 ├─ 맥미니 en1 (192.168.0.75) — Wi-Fi, 인터넷엔 미사용
                 └─ 맥북·그램·아이폰·아이패드 — Wi-Fi (인터넷 = AX3000BCM 경유)
```

- **맥미니 인터넷은 en0(상위 공유기 유선)** — AX3000BCM을 거치지 않음. AX3000BCM의 QoS·무선 설정은 맥미니 인터넷에 영향 없음.
- **무선 기기(맥북·그램·폰·패드)는 AX3000BCM 경유** — AX3000BCM 설정의 수혜자.

## 메인 공유기: ipTIME AX3000BCM

- 192.168.0.1 — 듀얼밴드 Wi-Fi 6 — 펌웨어 **15.34.0** (신형 `/ui/` 웹UI, SPA)
- 5GHz SSID `805_5G` (채널 36 / **80MHz**), 2.4GHz SSID `805` (채널 자동)
- 인증 **WPA2/WPA3 혼용** (WPA3SAE/WPA2PSK+AES)
- DNS **1.1.1.1 / 8.8.8.8** — DHCP로 전 클라이언트에 배포
- 포트포워드 0 / UPnP 규칙 0 / DMZ off / 원격관리 off
- QoS 꺼짐 — 2026-05-16 맥북 세션이 75/44+스마트QoS 적용 테스트 → 효과 없음(악화) → OFF 복원 (상세 `tasks/macbook.md`). ⚠ 적용 버튼 누르면 Down/Up 값이 ÷10 되는 펌웨어 버그 있음
- ⚠ 무선 설정 변경 시 WLAN 전체가 ~50-80초 재시작됨 (QoS 변경은 무선 재시작 없음)

## 회선

LG U+ 100M급 (실측 다운로드 ~86Mbps 천장). **라우터·Wi-Fi 튜닝으로 이 천장은 못 넘는다** — 본질적 개선은 LG U+ 회선 업그레이드(101 콜)뿐. 남은 튜닝은 지연(버퍼블로트·Wi-Fi 지터) 완화이지 속도 향상이 아님.
