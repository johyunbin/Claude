# 작업: 그램 네트워크 & 체감속도 최적화

- **대상**: 그램 (Windows 11, LG Gram, 고정 IP 192.168.0.50, Wi-Fi 연결, 사용자 `wh850`)
- **작성**: 맥미니 세션, 2026-05-16
- **수행 방법**: 그램 로컬 Claude Desktop 세션에서 진행 (RDP 원격제어 X — 키 입력 불안정)
- 먼저 `context/machines.md`를 읽을 것.

## 배경 / 목표

사용자 체감 두 가지:
1. 네트워크가 맥미니보다 버벅인다.
2. PC 자체가 좀 느리다.

100M 회선 자체는 못 바꾸므로, **그 한도 내에서** 네트워크 + 시스템 체감속도를 끌어올린다.

## 왜 맥미니보다 느리게 느껴지나 (1차 가설)

- 맥미니는 **유선 이더넷**으로 인터넷 → 저지연·안정.
- 그램은 **Wi-Fi**로 AX3000BCM 연결 → 본질적으로 지연·변동이 큼.
- → 유선 연결이 가능하면 그게 가장 큰 개선. 불가하면 Wi-Fi 품질을 최대로.

## 점검 & 개선 항목 (전부 비파괴)

### 네트워크
- [ ] Wi-Fi 밴드 — 반드시 5GHz(`805_5G`)에 붙어야 함. 2.4GHz(`805`)면 5GHz로 전환.
      `netsh wlan show interfaces` 로 라디오 종류·채널·수신/송신 속도·신호 확인.
- [ ] 링크 속도/신호 — 5GHz·160MHz면 송신률 1000Mbps+ 정상. 낮으면 위치·간섭·드라이버.
- [ ] Wi-Fi 드라이버 최신화 (제조사 최신 드라이버 — 체감 안정성에 큰 영향).
- [ ] DNS — 공유기가 1.1.1.1을 DHCP로 배포. 어댑터 DNS가 "자동"이면 그대로 OK.
      `Get-DnsClientServerAddress` 로 확인. 수동 오설정만 아니면 손댈 것 없음.
- [ ] NIC 절전 — 장치관리자 → Wi-Fi 어댑터 → 전원 관리 →
      "전원을 절약하기 위해 ... 끌 수 있음" **체크 해제** (지연·끊김 감소).
- [ ] `ipconfig /flushdns`.
- [ ] 유선 가능 여부 — 이더넷 포트/USB 동글이 있으면 유선이 최선.

### 체감 속도
- [ ] 시작 프로그램 — 작업관리자 → 시작 앱 → 불필요한 항목 "사용 안 함".
- [ ] 자원 점유 프로세스 — 작업관리자 프로세스 탭, CPU/메모리/디스크 정렬해 확인.
- [ ] 디스크 — SSD 여부, 여유 공간, 디스크 100% 점유 프로세스.
- [ ] Windows 업데이트 — 보류 중 업데이트 설치 (재부팅 필요할 수 있음).
- [ ] 전원 모드 — 설정 → 전원 → AC 연결 시 "최고의 성능".
- [ ] 한국 보안 SW(은행/공공용) — 키보드 후킹·백그라운드 상주가 체감 저하 원인일 수 있음.
      평소 비활성 + 필요 시 수동 시작 패턴 검토 (맥미니에서 동일하게 처리한 전례 있음).
- [ ] 시각 효과 — 필요 시 시스템 속성 → 성능 → "최적 성능".

## 이미 처리된 것

- AC 전원 시 화면끄기·절전 → "안 함" (사용자가 직접 설정함).
  → 덮개 닫을 때 동작도 "아무것도 안 함"인지 확인하면 절전 문제 완결 (전원·배터리 → 덮개 컨트롤).
- 공유기는 이미 최적화 완료 (펌웨어 15.34.0 / WPA3 / DNS 1.1.1.1 — `context/machines.md`).

## 주의

- 비파괴 작업만. 어댑터 비활성/재활성, `netsh winsock reset` 등 연결을 끊거나 재부팅이
  필요한 작업은 사용자에게 알리고 진행.
- **한/영 키 이슈**: 맥미니에서 RDP로 제어할 때 stuck modifier가 생겨 한/영 키가 먹지
  않은 적 있음. 재부팅하면 입력 상태가 초기화돼 해결됨.
